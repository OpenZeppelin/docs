Ownable is a lightweight authorization primitive: a single owner party is authorized for all privileged actions, and ownership can be transferred.

## Overview

Ownable models the common case where exactly one party controls the privileged surface of a workflow. The owner can perform restricted actions and can transfer ownership to another party. It is the smallest useful access primitive, and a natural default before a workflow grows enough to need the multiple roles of [Access Control](/canton/library/access-control).

## When to use it

Use Ownable when a single administrator is sufficient. If you need to authorize several parties or several kinds of action independently, use [Access Control](/canton/library/access-control) instead.
